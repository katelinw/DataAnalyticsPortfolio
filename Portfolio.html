Instructional Method Pass & Drop Rates

- I wrote a query that found pass rates & drop rates for first-time students broken out by instructional method & optionally student demographics
- I had to use lots of case statements to allow for proper filtering in the dashboard and lots of joins
- I put this into an interactive Tableau dashboard that allowed users to see pass rates and drop rates in each instructional method over the past five years in bar charts & trendlines
- This was also broken out by student demographics including race, gender, age group, & class standing to see which of the categories within those demographics have a higher pass rate
- Users were also able to look at this for specific courses using a filter and there was a dashboard tab for looking at developmental math & English as requested

Housing Credit Check

- I was given an Excel file with a list of student names & id's who were currently living in housing that I had to retrieve matching data for
- In Excel, I added a column for the insert statement so that I could use it for a temporary table
- I created a temporary table in the database & inserted the rows of the student's names & id's into it
- I then joined this table with the student table in our database on ID & selected the requested fields that corresponded to that student from the student table
- I also had to join that to the student history table so that I could calculate the total number of credits that a student is currently enrolled in because the purpose of this request was to make sure that all students who were currently in housing met the credit enrollment requirement
- I then exported this data result set directly from Toad as an Excel file & sent it to the stakeholder

Students with Holds

- I wrote a query that returned students who have holds on their status based on date, hold code selection, & release status sections which was written as a case statement
- I put this into Argos to create a report that allowed the user to select a date that would retrieve holds placed on or after it, select one or multiple hold types, & the option to select a release status
- This would then return all students who applied to the criteria. It would return student ID, name, hold code, hold description, the date the hold was placed, the date the hold was released, the release status, & who placed the code
- Argos uses variables in the SQL query that are implemented into the where statements which is what makes the filters work

Grads Who Were WSU Undergrads

- I was requested to find the percentage of students who are in the graduate nursing program for a selected term who previously attended WSU & who transferred to WSU specifically for the graduate nursing program
- I wrote the query for this that found grad students who were currently enrolled, joined this to the program of study table to return those current grad students with their department, & joined students who were previous WSU students by checking if they were actively enrolled with an undergraduate admit code
- This query primarily included window functions & left joins
- I then put it into Tableau to show the percentage in a bar chart & allow the user to select the term
- I also allowed the user to select the department so that this dashboard could be used for all departments rather than just nursing which was what this request revolved around. This way, it could be used by others & prevent further requests for other departments
